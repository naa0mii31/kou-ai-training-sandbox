# AGENTS.md

## 目的

このリポジトリは、家計簿アプリを作りながら Codex app と GitHub Flow を学ぶための練習環境です。

現場伴走スタッフが、受講者に「要件定義 -> Issue -> Codex実装 -> localhost確認 -> PRレビュー -> Merge判断」の流れを教えられるようにすることを重視します。

## 基本ルール

- 日本語で説明する。
- 1 Issue = 1 Branch = 1 PR を基本にする。
- Issue外の変更をしない。
- 実装前にDoDと「やらないこと」を確認する。
- 変更後は `npm run lint`、`npm test`、`npm run build` を実行する。
- PR本文には、変更内容、検証結果、リスク、ロールバック方法を書く。
- localhostで確認した内容をPR本文に残す。

## 禁止事項

- `.env*` を作成、閲覧、編集しない。
- APIキー、認証情報、個人情報、実顧客情報を扱わない。
- 本番API、本番DB、外部送信を扱わない。
- 金銭判断、契約変更、カード操作、振込操作を扱わない。
- mainへ直接pushしない。
- Issue外の親切修正を追加しない。

## Codex appで重視すること

- まずIssueの目的を読み、作業範囲を確認する。
- 実装前に短い計画を出す。
- 差分は必ず人間が読む。
- 動いたかどうかをlocalhostで確認する。
- AIの出力をそのまま信用せず、自分の言葉で説明できる状態にする。

## 完了条件

- IssueのDoDを満たした。
- テストとbuildが通った。
- localhostで動作確認した。
- 差分を説明できる。
- 不要な変更がない。
- 禁止事項に触れていない。
