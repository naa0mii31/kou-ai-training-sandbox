# Definition of Done

Issueを完了と判断する前に、以下を確認します。

## 機能

- [ ] Issueの目的を満たしている
- [ ] Issue外の変更をしていない
- [ ] 今回やらないことに触れていない
- [ ] localhostで操作して確認した

## 検証

- [ ] 必要なテストを追加または更新した
- [ ] `npm run lint` が通った
- [ ] `npm test` が通った
- [ ] `npm run build` が通った

## PR

- [ ] PR本文に変更内容を書いた
- [ ] PR本文に検証結果を書いた
- [ ] リスクや未確認点を書いた
- [ ] ロールバック方法を書いた
- [ ] 禁止事項に触れていない

## 伴走者の確認

- [ ] 受講者が差分を自分の言葉で説明できる
- [ ] localhostと本番環境の違いを説明できる
- [ ] 次に作るIssueが1つ決まっている
