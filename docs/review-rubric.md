# Review Rubric

PRを見るときは、次の順番で確認します。

## 1. Issue適合

- Issueの目的を満たしているか
- DoDを満たしているか
- Issue外の変更がないか
- やらないことに触れていないか

## 2. 家計簿アプリとして自然か

- 入力しやすいか
- 一覧で見やすいか
- 合計やカテゴリが期待通りか
- 受講者の要件に合っているか

## 3. 実装

- 変更が小さいか
- 読みやすいか
- 不要な抽象化がないか
- 自分の言葉で説明できるか

## 4. テスト

- 必要なテストがあるか
- 回帰を防げるか
- `npm run lint`、`npm test`、`npm run build` が通っているか

## 5. 安全性

- `.env*` や認証情報に触れていないか
- 本番APIや外部送信に触れていないか
- 実データや個人情報が入っていないか
- 金銭判断や契約変更に関わっていないか

## 6. Merge判断

- レビューコメントが解決済みか
- CIが通っているか
- localhostで確認済みか
- 戻し方が書かれているか
