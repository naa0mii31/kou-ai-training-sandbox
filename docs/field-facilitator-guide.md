# 現場伴走スタッフ向けガイド

## 役割

現場伴走スタッフは、開発者になる人ではありません。受講者がAIとGitHubを使って小さな改善を進められるよう、横で支える人です。

## 最終ゴール

4時間研修の準担当として、次の流れを進行できる状態を目指します。

1. 受講者にAIとの要件定義を体験してもらう
2. 家計簿アプリのゴールを1枚にまとめる
3. Issueへ小さく分ける
4. Codex appで実装を依頼する
5. localhostで動作確認する
6. PRの差分とテスト結果を一緒に見る
7. 本番環境との違いを説明する
8. 次のIssueを決める

## 4時間研修の流れ

| 時間 | やること | 残すもの |
|---:|---|---|
| 0:00-0:20 | ゴール共有、安全ルール | 本日のゴール |
| 0:20-0:50 | AIに3つ質問してもらう | 要件メモ |
| 0:50-1:30 | Issueへ分解する | Issue案 |
| 1:30-2:30 | Codex appで1 Issue実装 | PRまたは差分 |
| 2:30-3:10 | localhostで確認 | 動作確認メモ |
| 3:10-3:40 | PRレビュー | レビューコメント |
| 3:40-4:00 | 次回Issueと宿題 | 次アクション |

## 伴走者が判断してよいこと

- 文言の軽微な修正
- Issueの粒度調整
- Codexへの依頼文の改善
- localhost確認の補助
- PR本文の不足指摘

## 持ち帰るべきこと

- 実データや個人情報を扱う必要が出た
- 本番環境へ公開したい
- 外部APIやAPIキーが必要になった
- 契約、金銭、医療広告、法務判断が関わる
- AIの差分を誰も説明できない

## 訪問後レポートの観点

- 受講者が自分でできたこと
- 詰まったところ
- 説明が必要だった言葉
- 次回までの宿題
- Issue化できそうな改善点
- 安全上の懸念
