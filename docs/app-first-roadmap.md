# App-first 学習ロードマップ

## 方針

先に家計簿アプリを作ります。GitHub、Codex、Issue、PR、テスト、localhost、本番環境は、必要になったタイミングで学びます。

## 6週間カリキュラム

| 週 | テーマ | 作るもの | 学ぶこと |
|---:|---|---|---|
| 1 | AIと要件定義 | 自分用家計簿の要件メモ | AI壁打ち、MVP、やらないこと |
| 2 | Issue分解 | GitHub Issue 5から8本 | Issue、DoD、変更範囲 |
| 3 | 最初の実装 | 支出入力と一覧表示 | Codex app、PR、差分確認 |
| 4 | localhost確認 | 動く家計簿v1 | 開発環境、エラー確認、検証メモ |
| 5 | 便利機能 | 集計、保存、編集、削除 | テスト、CI、レビュー対応 |
| 6 | 自分専用機能 | 予算、アラート、グラフなど | 追加要件、Issue再分解、Merge判断 |

## 学習ステップ

### Lv0: 安全ルール

- 実データを使わない
- `.env` やAPIキーを扱わない
- 本番APIや外部送信に触れない

### Lv1: 作りたいものを決める

- AIに3つだけ質問してもらう
- 自分の生活に合う家計簿のゴールを決める
- 今回やらないことを決める

### Lv2: Issueに分ける

- 1 Issue = 1つの小さな機能
- DoDを書く
- やらないことを書く

### Lv3: Codex appで実装する

- Issueを読ませる
- 変更範囲を指定する
- PR本文まで書かせる

### Lv4: localhostで確認する

- `npm run dev` で起動する
- ブラウザで操作する
- 動作確認メモを残す

### Lv5: PRレビューする

- 差分を見る
- テスト結果を見る
- リスクとロールバックを見る
- Mergeしてよいか判断する

### Lv6: 現場で教える

- 受講者の画面を見ながら詰まりを拾う
- 判断が必要なことは持ち帰る
- 訪問後レポートを書く
