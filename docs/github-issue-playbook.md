# GitHub Issue Playbook

## Issueとは

Issueは「やること」を小さく切ったものです。家計簿アプリでは、1つのIssueで1つの機能だけを扱います。

## 良いIssueの型

```md
## 目的

なぜこの機能を作るのか。

## やること

- 具体的に変更すること

## 完了条件 DoD

- 何ができたら完了か

## やらないこと

- 今回は触らないこと

## 確認方法

- localhostで何を確認するか
- どのテストを実行するか
```

## 家計簿アプリのIssue例

1. 支出入力フォームを作る
2. 支出一覧を表示する
3. カテゴリを追加する
4. 月別フィルタを追加する
5. カテゴリ別合計を表示する
6. localStorageに保存する
7. 編集と削除をできるようにする
8. READMEにlocalhost確認手順を書く

## 伴走者の声かけ

- 「このIssueで今日作るものは何ですか？」
- 「完了条件はどれですか？」
- 「今回はやらないことは何ですか？」
- 「localhostでは何を確認しますか？」
- 「PRで何をレビューしてほしいですか？」
