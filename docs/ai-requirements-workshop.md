# AIと要件定義するワークショップ

## 目的

受講者が「自分に合った家計簿アプリ」をAIと一緒に決めます。最初から完璧な仕様を作るのではなく、最初に動くMVPを決めます。

## AIに渡す最初の依頼

```text
あなたにぴったりの家計簿アプリを作りたいです。
良いアウトプットを出すために、私に聞きたいことを3つに絞って質問してください。
質問は1つずつ行ってください。
```

## AIに聞いてもらう3問

1. 毎月いちばん見えるようにしたいお金の不安や目標は何ですか？
2. いつ・どこで家計簿を入力したいですか？スマホ中心、PC中心、週末まとめ入力などを教えてください。
3. 技術構成はどうしますか？推奨はNext.jsです。もっと軽くするならHTML/CSS/JavaScript、Reactだけを学びたいならVite + Reactも選べます。

## 要件定義メモの型

```md
# 私の家計簿アプリ要件

## アプリ名

## 使う人

## 解決したい悩み

## 最初に必ずほしい機能

## あるとうれしい機能

## 今回やらないこと

## 完成条件

## 技術構成
```

## 伴走者の注意

- 受講者の答えを否定しない。
- 機能を増やしすぎない。
- 最初は「入力できる」「一覧で見える」「合計が出る」を優先する。
- 銀行口座、カード情報、実名、住所などは入れない。
